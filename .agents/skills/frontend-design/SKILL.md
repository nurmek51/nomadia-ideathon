This skill guides creation of distinctive, production-grade frontend interfaces that feel designed, not autogenerated. The goal is memorable execution with clear hierarchy, technical discipline, and product credibility.

The user provides frontend requirements: a component, page, application, or interface to build. They may include context about the purpose, audience, brand, or technical constraints.

## Design Thinking

Before coding, understand the context and commit to a clear direction:

- **Purpose**: What job does this interface need to do well?
- **Product posture**: Is this editorial, utilitarian, expressive, competitive, playful, premium, or institutional?
- **Constraints**: Framework, accessibility, performance, responsiveness, interaction density.
- **Differentiation**: What single visual or interaction idea will make this specific product memorable?

Choose an intentional direction and execute it precisely. Distinctive work does not require chaos. Restraint is valid if it is specific, branded, and well-crafted.

Then implement working code that is:

- Production-grade and functional
- Cohesive with a clear aesthetic point of view
- Visually memorable without compromising usability
- Refined at the level of spacing, typography, motion, and states

## Frontend Aesthetics Guidelines

Focus on:

- **Typography**: Use fonts with character and strong hierarchy. Avoid generic defaults such as Arial, Inter, Roboto, or anonymous system stacks unless the existing product already uses them.
- **Color & Theme**: Use CSS variables and a small, controlled palette. Strong neutrals plus one or two assertive accents usually outperform busy rainbow systems.
- **Motion**: Use a few meaningful motion moments rather than layering generic micro-animations everywhere. Motion must support feedback, focus, and emotional tone.
- **Composition**: Prefer layouts with an obvious anchor and supporting regions. Asymmetry is allowed only when it improves the page's identity without damaging scanability.
- **Surface Treatment**: Build atmosphere through material cues, borders, textures, lighting restraint, and spacing. Avoid AI-default gradients, glass panels, glow haze, or decorative filler.

## Guardrails

- Never default to generic AI-generated aesthetics.
- Never use spectacle to cover weak information architecture.
- Never apply the same visual trick to every component.
- Never sacrifice mobile usability for desktop drama.
- Never treat landing pages and application shells as the same design problem.

## Product UI Guidance

For product interfaces:

- make the primary workflow visually dominant
- keep navigation and controls plain enough to disappear when used
- let dense information feel calm through spacing and hierarchy, not emptiness
- use one memorable idea per page, not ten competing ones

For marketing or showcase pages:

- allow larger visual gestures
- still preserve believable product structure
- keep calls to action grounded in the product's actual value

## Complexity Rule

Match implementation complexity to the concept:

- minimal or editorial directions need precision, not decoration
- maximal or expressive directions need coherent systems, not random effects
- competitive or utility-focused products should privilege clarity under pressure

## Output Standard

When building a page or component, the result should feel:

- branded
- intentional
- reviewable by a serious designer without embarrassment
- implementable and maintainable by engineers
